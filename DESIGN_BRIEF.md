# Design Brief — KMO Rack Bar Custom Landing Page (สำหรับ Google Stitch / นักออกแบบ)

**อ้างอิงภาพสไตล์:** โปสเตอร์สินค้า KMO (แนวทัวร์ริ่ง ดำ+เหลือง สไตล์อุตสาหกรรม/หน้างาน)
**สิ่งที่ต้องการจากภาพ:** เอาเฉพาะ "ดีไซน์/โทน/เลย์เอาต์" — ไม่เอารูปรถ/รูปสินค้าในภาพมาใช้จริง

---

## 1. ทิศทางดีไซน์ใหม่ (จากภาพอ้างอิง)

| องค์ประกอบ | รายละเอียด |
|---|---|
| โทนสีหลัก | พื้นหลังดำสนิท + เหลืองสด (safety yellow เข้มข้น ไม่ใช่ทองหรูหรา) + ขาว |
| ความรู้สึก | อุตสาหกรรม / หน้างาน / "ใช้จริง ไม่ต้องแปลง" — ตรงข้ามกับแนว "หรูหรา premium" |
| ตัวอักษรหัวข้อ | ฟอนต์หนา ตัวเอียง/ตั้งตรงบล็อกแน่น พิมพ์ใหญ่ทั้งหมด (all-caps) เน้นความหนักแน่น |
| แบนเนอร์/ป้าย | รูปทรงกระดาษฉีก/ปาดแปรง (brush-stroke) สีเหลืองทับข้อความ ใช้เน้นคำสำคัญ |
| เส้นแบ่ง section | ลายทางเฉียงดำ-เหลือง (hazard stripe) คั่นระหว่างบล็อกเนื้อหา |
| Feature/คุณสมบัติ | กล่องไอคอนมีกรอบเหลือง (icon + หัวข้อสั้น + คำอธิบาย 1 บรรทัด) จัดเป็นแถว/กริด |
| รายการสินค้า/checklist | bullet เป็นเครื่องหมายถูกสีเหลือง |
| Footer | แถบดำ มีโลโก้ + สโลแกน + ช่องทางติดต่อ (โทร/Facebook) ในแถวเดียว |

**⚠️ จุดที่ต้องตัดสินใจ:** ทิศทางนี้ต่างจาก brand guideline เดิมที่บันทึกไว้ใน `CLAUDE.md` ของโปรเจกต์ (Premium Dark Theme สีทองหรูหรา #e8b84b, ฟอนต์ Kanit/Sarabun) พอสมควร — เป็นการเปลี่ยนทิศทางแบรนด์ทั้งเว็บ ไม่ใช่แค่ปรับสี ควร confirm กับ CEO ให้ชัดว่าจะแทนที่แนวเดิมทั้งหมด หรือใช้เฉพาะบาง section (เช่น hero/โปรโมชั่น) แล้ว section อื่นคงธีมเดิมไว้

## 2. Site Map ปัจจุบัน (โครงสร้างจริงของเว็บ ตอนนี้)

| Section | ทำหน้าที่ |
|---|---|
| Header/Nav | โลโก้ + เมนู (หน้าแรก/แคตตาล็อก/จองคิว) + ปุ่มตะกร้า |
| Hero | หัวข้อใหญ่ + คำโปรย + ปุ่ม CTA 2 ปุ่ม (ดูสินค้า/เช็คคิว) |
| Catalog | ช่องค้นหา + ปุ่ม filter หมวดหมู่ (6 หมวด+ทั้งหมด) + grid การ์ดสินค้า (ดึงจาก Google Sheets CSV แบบ real-time) |
| Booking Info | บล็อกอธิบายเงื่อนไขจองคิว (เวลาทำการ/มัดจำ/ความสะดวก) + ปุ่ม CTA |
| Cart Drawer | แผงเลื่อนออกจากขวา แบ่ง 2 กลุ่ม (จองคิวติดตั้ง / สั่งซื้อกับทางร้าน) แยกยอดรวม+ปุ่มเช็คเอาต์ |
| Footer | เวลาทำการ, ช่องทางติดต่อ (LINE/เบอร์โทร), ลิงก์ GitHub |

## 3. Element ที่ห้ามเปลี่ยนชื่อ/ลบ (JS ผูกอยู่ ถ้าเปลี่ยนฟีเจอร์จะพัง)

ดีไซน์ใหม่ปรับหน้าตา/สี/เลย์เอาต์ได้เต็มที่ แต่ **id/class ต่อไปนี้ต้องคงไว้เป๊ะๆ** (เปลี่ยนสไตล์ได้ เปลี่ยนชื่อไม่ได้):

- `#product-grid-container` — ที่ใส่การ์ดสินค้า (JS generate เข้ามาเอง)
- `#catalog-search-input` — ช่องค้นหา
- `.filter-btn` + attribute `data-category` — ปุ่มกรองหมวด (ต้องมีครบ 7 ปุ่ม: all/rear/side/crashbar/accessory/gear/service)
- `#cart-toggle`, `#cart-drawer`, `#cart-drawer-backdrop`, `#cart-close` — เปิด/ปิดตะกร้า
- `#cart-items-container` — list รายการในตะกร้า
- `#cart-total-val`, `#cart-deposit-val`, `#cart-balance-val` — ยอดฝั่งจองคิว
- `#btn-checkout-cart` — เช็คเอาต์ฝั่งจองคิว
- `#btn-checkout-order` — เช็คเอาต์ฝั่งสั่งซื้อ
- `#cart-badge-count` — ตัวเลขนับของในตะกร้าบนปุ่ม

ถ้า Stitch/ดีไซเนอร์ไม่ทราบเรื่อง id พวกนี้ ให้ export มาเป็น "ดีไซน์อย่างเดียว" (สี/ฟอนต์/เลย์เอาต์/รูปทรง) แล้วให้ทีมพัฒนา (Codex) เป็นคนแปะกลับเข้าโครงสร้างเดิม ไม่ต้อง generate โค้ดที่ทำงานได้จริงมาตรง ๆ
